# Script Mode

## Goal
Run a SQL file through toydb without interactive input.

## Behavior

- Read the whole file.
- Split statements on `;`.
- Skip empty statements.
- Execute statements in order.
- Stop on the first error.
- Print each statement result as it is produced.

## CLI

```text
toydb -f script.sql
```

## Why this exists

It gives a quick way to replay a small SQL scenario and see the current
state of the tokenizer, parser, catalog, storage, and query output without
typing statements manually.
